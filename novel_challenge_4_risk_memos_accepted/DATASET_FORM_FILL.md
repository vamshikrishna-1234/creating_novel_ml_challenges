# Dataset Description

## Overview

This dataset is synthetically generated (no real supply chains, companies, or events). It represents multi-section **risk memos** following a fictional reporting protocol called **ORBIT** (Operational Risk Briefing & Impact Taxonomy). Each row is one memo describing a potential disruption in a fictional supply network. The task is to predict the **disruption severity tier** (ordinal 5-class classification).

**Dataset Summary:**

- **Multi-section documents:** Each memo has 2–5 ORBIT sections (e.g. `[ORBIT:VECTOR]`, `[ORBIT:IMPACT]`, `[ORBIT:MITIGATION]`, `[ORBIT:CASCADE]`, `[ORBIT:NOTE]`) with English narrative, risk codes (e.g. ORB-V3, ORB-I5, ORB-M2), and numeric estimates. An `[ORBIT:NOTE]` distractor section with tier-irrelevant analyst boilerplate appears in ~40% of memos.
- **Severity tiers:** `tier_1_minor`, `tier_2_moderate`, `tier_3_significant`, `tier_4_severe`, `tier_5_critical` — determined by weak, overlapping combinations of narrative phrasing, shortfall magnitude, cascade depth, and mitigation effectiveness. Narrative templates are shared across tiers (same phrase can appear at any tier); numeric ranges overlap heavily.
- **Structured features:** `region` (6 fictional regions) and `commodity_class` (8 fictional commodity types) accompany each memo.
- **Reproducibility:** Data is produced by `generate.py` v2 (default seed 42, 32,000 rows). Python standard library only; no external sources.

**Current raw dataset stats:**

- Total rows: 32,000
- Columns: id, memo, region, commodity_class, label
- Label distribution: tier_1_minor ~28%, tier_2_moderate ~24%, tier_3_significant ~22%, tier_4_severe ~16%, tier_5_critical ~10%
- Regions: 6, roughly balanced (~5,300 each)
- Commodities: 8, roughly balanced (~4,000 each)
- Memo length: 287–617 characters

## File Structure

- `data.csv` — labeled data (id, memo, region, commodity_class, label)

No other files in the raw dataset. The challenge prepare script splits this into public/private splits.

## Features

| Column          | Type   | Description |
|-----------------|--------|-------------|
| id              | int    | Unique identifier |
| memo            | string | Multi-section ORBIT risk memo. Contains ORBIT section headers, risk codes (ORB-V/I/M), and English narrative. |
| region          | string | Fictional region. One of: norath, veldan, crymara, esthos, lunavar, torwen. |
| commodity_class | string | Fictional commodity. One of: synth-ore, bio-compound, flux-crystal, plasma-stock, neuro-fiber, cryo-element, photon-alloy, gravity-mesh. |
| label           | string | Severity tier. One of: tier_1_minor, tier_2_moderate, tier_3_significant, tier_4_severe, tier_5_critical. |

## Notes

- **Reproducibility:** Run `python generate.py --output data.csv --seed 42 --size 32000` to regenerate.
- **ORBIT protocol:** Invented structured reporting sections and risk codes carry weak, overlapping signal about severity. Narrative templates are shared across tiers with different frequency weights; numeric ranges overlap heavily. The raw data includes an `[ORBIT:VERDICT]` section that explicitly states the tier; the challenge prepare script strips this section so solvers must infer from the remaining content.
- **One label per row:** Each id appears exactly once. No duplicate memos in the generator output.
- **Class imbalance:** Intentionally imbalanced (tier_5_critical is ~10% minority).
- **Train/test split:** The challenge prepare script splits by row (80/20) with stratification and a fixed seed; no overlapping ids between train and test.
