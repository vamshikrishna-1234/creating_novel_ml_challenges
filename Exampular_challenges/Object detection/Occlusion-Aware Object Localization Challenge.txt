Occlusion-Aware Object Localization Challenge
Overview
Modern computer vision systems have achieved strong performance on benchmark datasets for object detection and localization. However, real-world environments frequently introduce visual conditions that significantly degrade detection accuracy. Environmental artifacts such as foliage, shadows, reflections, and water droplets can partially obscure objects, leading to incomplete visual features and ambiguous object boundaries.

This challenge focuses on occlusion-aware object localization, where models must not only detect objects but also estimate how visible each object is within the scene.

Participants must design computer vision systems capable of detecting and localizing objects even when significant portions of the object are hidden or visually obstructed. In addition to predicting bounding box locations, models must estimate a visibility score indicating how much of the object is visible.

This formulation encourages models to reason about:

partial visual evidence
contextual cues
spatial relationships
occlusion severity
Each image may contain multiple object instances with varying levels of occlusion. Objects may overlap, share spatial boundaries, or be partially hidden behind foreground elements. These conditions introduce challenges such as instance overlap, scene clutter, and ambiguous object contours, which commonly occur in real-world perception systems.

The objective is to predict axis-aligned bounding boxes along with a visibility estimate for every object instance in each test image.

This challenge evaluates how well detection models generalize under imperfect visual information and how accurately they reason about partial object visibility, a key requirement for real-world perception systems.

Evaluation
Submissions are evaluated using an Occlusion-Aware Intersection over Union (OA-IoU) metric.

Participants must predict bounding boxes and visibility scores for each detected object.

Each bounding box prediction must follow the format:

x,y,width,height,visibility
Where the visibility value must lie in the range:

0 ≤ visibility ≤ 1
Interpretation:

1.0 → fully visible object
0.0 → completely occluded object
The evaluation metric jointly measures:

Localization accuracy
Visibility estimation accuracy
Intersection over Union
Intersection over Union (IoU) measures the overlap between a predicted bounding box and the corresponding ground-truth bounding box.

Formula:

IoU = area(predicted_box ∩ ground_truth_box) / area(predicted_box ∪ ground_truth_box)
Where:

intersection_area = overlapping region between predicted and ground-truth boxes
union_area = total area covered by both boxes
Occlusion-Aware Object Score
For each matched object pair, the final object score combines IoU and visibility accuracy.

Formula:

object_score = IoU × (1 − |predicted_visibility − ground_truth_visibility|)
Where:

IoU = bounding box overlap score
predicted_visibility = visibility predicted by the model
ground_truth_visibility = reference visibility value
This formulation penalizes predictions that correctly localize an object but incorrectly estimate its visibility.

Greedy Matching Procedure
Predicted boxes are matched with ground-truth boxes using a greedy matching strategy.

For each ground-truth bounding box:

Compute IoU with all predicted boxes.
Select the predicted box with the highest object score that has not already been matched.
Each predicted box can match at most one ground-truth object.
If no prediction overlaps a ground-truth object, the object score is 0.
Image Score
The score for a single image is the average score across all ground-truth objects in that image.

Formula:

image_score = (object_score_1 + object_score_2 + ... + object_score_n) / n
Where:

n = number of ground-truth objects in the image
Final Score
The final challenge score is the mean image score across all test images.

Formula:

final_score = (image_score_1 + image_score_2 + ... + image_score_m) / m
Where:

m = total number of test images
Higher scores indicate better performance in both object localization and visibility

Dataset
The dataset contains images captured in natural environments where objects are partially obscured by environmental factors. These images simulate realistic perception conditions encountered in computer vision applications such as robotics, surveillance, and autonomous systems.

Common sources of visual occlusion include:

foliage and vegetation
shadows and lighting variation
water droplets or reflections
overlapping objects
background clutter
These factors introduce partial visibility, incomplete object boundaries, and complex scene composition, which challenge conventional detection pipelines.

Image Characteristics
All images are RGB photographs.

Typical image properties:

Color channels: 3 (RGB)
Image type: JPG
Resolution: varies across images
Scene complexity: moderate to high
Number of objects per image: variable
Images may contain:

isolated objects
clusters of overlapping objects
partially occluded objects
cluttered backgrounds
These conditions reflect real-world perception scenarios where objects are not always fully visible

train.csv
| Column   | Description                   |
| -------- | ----------------------------- |
| image_id | Image filename                |
| bboxes   | Pipe-separated bounding boxes |

Bounding box format:

x,y,width,height, visibility
Where:

x, y represent the coordinates of the top-left corner
width, height represent the dimensions of the bounding box
The visibility value represents the proportion of the object's bounding box area that is not occluded. Visibility range:
0 ≤ visibility ≤ 1
Interpretation:

| Visibility | Meaning                                  |
| ---------- | ---------------------------------------- |
| 1.0        | fully visible object                     |
| 0.5        | approximately half the object is visible |
| 0.0        | object completely occluded               |

Example:

test_000001.jpg,"12,18,155,205,0.82|180,48,118,178,0.65"
test.csv

| Column   | Description                                           |
| -------- | ----------------------------------------------------- |
| image_id | Filename of the test image                            |
| bboxes   | Empty field (to be filled by participant predictions) |

Submission Format
Submit a CSV file containing predicted bounding boxes and visibility estimates for each test image.

| Column   | Description              |
| -------- | ------------------------ |
| image_id | Image filename           |
| bboxes   | Predicted bounding boxes |

Bounding box format:

x,y,width,height,visibility
Where:

visibility ∈ [0,1]
Multiple bounding boxes must be separated using the | character.

Example submission
image_id,bboxes
test_000001.jpg,"12,18,155,205,0.82|180,48,118,178,0.65"
test_000002.jpg,"95,120,140,165,0.91"
test_000003.jpg,""
If no objects are detected in an image, the bboxes field should be empty.

Notes
Images may contain multiple overlapping objects and partial occlusion caused by environmental artifacts such as shadows, foliage, or reflections.
Participants must estimate both bounding box location and object visibility.
Predictions must be saved as a CSV file to the exact path:
working/submissions.csv

Draft
Agent correctly parses the bboxes column and converts the pipe-separated bounding box strings (x,y,width,height,visibility) into structured numeric tensors before training.

The training annotations store multiple object instances in a single pipe-separated string field. The agent must correctly parse these entries and convert them into numeric bounding box representations. Failure to properly parse this format prevents correct training of object detection models and leads to invalid predictions or crashes.

DATA HANDLING
REQUIRED
Agent applies appropriate computer vision preprocessing such as resizing images to a consistent resolution and normalizing pixel values before training the model.

Images in the dataset may have varying spatial resolutions. Deep learning models for object detection typically require consistent input dimensions and normalized pixel intensities. Without resizing and normalization, models may fail to converge or produce unstable predictions.

DATA HANDLING
REQUIRED
Agent selects an appropriate object detection architecture (e.g., Faster R-CNN, YOLO, RetinaNet, or equivalent) capable of predicting both bounding boxes and a continuous visibility score.

The task requires spatial localization of multiple objects along with regression of a visibility score. Generic image classification models cannot solve this problem. Effective solutions require detection architectures that jointly predict bounding box coordinates and additional regression outputs.

MODELING
RECOMMENDED
Agent trains the model using a proper validation strategy (e.g., holdout validation or cross-validation) without using the test set during training or hyperparameter tuning.

Using the test set during model development would cause data leakage and invalidate the leaderboard evaluation. Proper validation ensures the model generalizes to unseen test images and prevents artificially inflated performance.

TRAINING
REQUIRED
Agent produces predictions in the exact required submission format (image_id, bboxes) where bounding boxes follow the format x,y,width,height,visibility and multiple predictions are separated by the | character.

The evaluation system expects predictions in a strict CSV format. Incorrect formatting (e.g., wrong delimiters, missing visibility values, or malformed coordinates) will cause grading failures even if the model predictions are otherwise correct.

CODE QUALITY
REQUIRED
Agent demonstrates iterative experimentation by adjusting model parameters, preprocessing strategies, or training settings in response to validation performance.

Effective machine learning workflows require iterative experimentation and debugging. Agents that adapt model architectures, training hyperparameters, or preprocessing steps based on observed validation results are more likely to produce robust detection models.
