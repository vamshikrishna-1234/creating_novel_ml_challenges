TikZ Code Generation Dataset
Overview
This dataset contains TikZ LaTeX source code paired with rendered diagram images and descriptive captions, derived from the DaTikZ-v2 corpus (https://huggingface.co/datasets/nllg/datikz-v2). Each entry represents a diagram commonly found in academic publications, including flowcharts, mathematical plots, circuit diagrams, geometric constructions, and schematic illustrations. The TikZ code has been filtered to entries between 100 and 5000 characters in length, keeping moderately complex diagrams suitable for code generation tasks.

File Structure
- raw/ -- Top-level directory containing all data
  - raw.csv -- Maps image filenames to captions, TikZ code, and split assignments
  - images/ -- Rendered diagram PNG images

Recovering Lost LaTeX Diagrams From Corrupted Source Files
Overview
A university research archive has suffered a partial disk failure. Thousands of LaTeX source files for academic papers were recovered, but the recovery was imperfect while document preambles, package imports, and environment declarations came through intact, many of the drawing commands inside TikZ diagram environments were corrupted beyond repair. Each corrupted region has been replaced with a [MASK] placeholder.

Fortunately, the compiled PDF figures survived on a separate backup drive, so the lab has a rendered PNG image and the original figure caption for every diagram. Your task is to act as an automated diagram source reconstructor: given a partially corrupted LaTeX document, the rendered target image, and the figure caption, produce the complete restored TikZ source code.

This is harder than generating TikZ from scratch; you must produce code that is consistent with the surviving fragments (package choices, coordinate systems, styling conventions) while inferring the missing drawing logic from the visual output.

The dataset provides 3000 training examples where both the corrupted and complete source are available. For the test set, only the corrupted source, images, and captions are provided. Predict the full restored code for each test document.

Evaluation
Submissions are scored using TikZ Structural Similarity (TSS), a composite metric that evaluates both token-level accuracy and structural command fidelity. The final score is the average per-example TSS across all test examples. Higher scores are better, with a maximum of 1.0.

`TSS = 0.6 * LaTeX-BLEU-4 + 0.4 * Command-F1`  
LaTeX-BLEU-4 is a BLEU-4 score computed with LaTeX-aware tokenization. Instead of whitespace splitting, the tokenizer splits on LaTeX command boundaries (\draw, \node, etc.), structural delimiters ({, }, (, ), ;), numeric values, and TikZ path operators (--, -|). Smoothing is applied to handle zero n-gram counts.

Command-F1 is the F1 score computed over LaTeX command multisets. All command invocations (e.g., \draw, \usepackage, \node) are extracted from both the reference and predicted code, and F1 is computed with multiplicity (command frequency matters, not just presence).
