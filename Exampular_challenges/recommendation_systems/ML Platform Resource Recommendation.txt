Overview
An open-source ML platform hosts thousands of resources -- models, datasets, and interactive applications (spaces) -- used daily by ML practitioners worldwide. Users interact with these resources by starring, downloading, and running them.

In this challenge you will build a recommendation system for platform resources. Given a user's past interaction history with models, datasets, and spaces, predict which resources they will interact with next. This is a cross-type recommendation problem: your system must recommend across three different resource types simultaneously.

The dataset captures real user-resource interaction patterns along with auxiliary signals from a knowledge graph (publishing relationships) and a social trust network.

Dataset
The dataset contains user-resource interactions from an ML platform ecosystem, along with auxiliary graph-structured data.

Dataset Statistics
Users: 38,624
Items: 25,080 (16,200 models, 5,246 spaces, 3,634 datasets)
Training interactions: approximately 867,000
Test users: 38,624
Knowledge graph triples: approximately 25,000
Social network edges: approximately 84,900
Item categories: 52
Interaction sparsity: 99.91%
File descriptions
train.csv - Training interactions between users and resources
test.csv - User IDs to generate predictions for
items.csv - Resource metadata (type information)
sample_submission.csv - Example of the expected submission format
kg_triples.csv - Knowledge graph triples capturing publishing relationships between entities
social_network.csv - Social trust/follow edges between users
categories.csv - Item-to-category assignments (items can belong to multiple categories)
Data fields
train.csv

user_id (int) - Anonymous integer identifier for a user
item_id (int) - Integer identifier for a resource the user interacted with
test.csv

user_id (int) - Anonymous integer identifier for a user who needs predictions
items.csv

item_id (int) - Integer identifier for a resource
item_type (str) - One of: model, dataset, or space
kg_triples.csv

head_entity_id (int) - Head entity identifier in the knowledge graph
relation (str) - Relationship type between entities (e.g., publishing relationships between users/organizations and resources)
tail_entity_id (int) - Tail entity identifier in the knowledge graph
social_network.csv

user_id_a (int) - Source user identifier
user_id_b (int) - Target user identifier (trust/follow relationship)
categories.csv

item_id (int) - Resource identifier matching items.csv
category_id (int) - Category label (0-51); items may belong to multiple categories
sample_submission.csv

user_id (int) - User to recommend for
predicted_items (str) - Space-separated list of exactly 10 item_id values, ranked best-first
