Chart Grounding Challenge (SCGC)

## 1. Overview
Modern Vision-Language Models (VLMs) often **hallucinate** numerical values or entities when reasoning over complex charts. These hallucinations can be "Direct Contradictions" (stating a value different from the chart) or "Ungrounded Claims" (making assertions that cannot be verified from the provided data).

### 1.1 Data Context & Source
The charts in this challenge are derived from a benchmark dataset for visual reasoning. 
- **Domains**: Data spans finance, sociology, public health, and climate science. 
- **Generation**: The "base responses" are generated by frozen LLMs/VLMs and then systematically perturbed to introduce grounded hallucinations based on visual and textual misinterpretations.

### 1.2 The Challenge
In the **Surgical Chart Grounding Challenge (SCGC)**, participants must build a "Grounding Layer". Unlike standard VQA which just asks for an answer, this challenge requires an agent to:
1. **Analyze** a pre-generated "base response" (which is known to be flawed).
2. **Detect** exact character spans of hallucinations.
3. **Attribute** specific evidence snippets from supplementary context that justify the facts.
4. **Surgically repair** the response with minimal edits to make it factually accurate.

---

## 2. Internal Mechanics & Rationale

### 2.1 Why Context Sentences?
While the chart image is the primary source of truth, real-world systems often use **Multimodal RAG**. Context snippets are provided to mimic auxiliary information (like OCR output, data tables, or associated reports). Your grounding layer must reconcile the image with this text.

### 2.2 The (Trap) Snippet
Some records contain a context snippet tagged with `(Trap)`. This is a "distractor" sentence that might contain keywords from the question but is factually irrelevant or misleading. 
> [!WARNING]
> Selecting a `(Trap)` snippet in your `evidence_snippets` will result in a **0.0 score** for that record's attribution. This forces models to perform true semantic grounding rather than simple keyword matching.

### 2.3 Predicted Fields
- `detected_hallucinations`: Participants must track *why* a model failed. Identifying the span is the first step to correction.
- `evidence_snippets`: Attribution ensures the "repair" isn't just another hallucination.

### 2.4 Multimodal Reasoning
The challenge is fundamentally **multimodal**. Participants are encouraged to use:
1.  **Visual Features**: Extracting data points directly from `image_raw` using OCR or patch-level embeddings.
2.  **Textual Context**: Leveraging the `[S1]...` snippets which serve as "grounded evidence" or auxiliary metadata.
A successful "Grounding Layer" cross-references both sources to identify where the `base_response` deviates from the visual and textual truth.

---

## 3. The Task
For each record, you are provided with:
- **question** (`string`): A natural language query about the chart.
- **image_raw** (`bytes`): Embedded PNG bytes of the chart.
- **context** (`string`): Snippet-tagged text in the format `[S1] ... [S2] ...`.
- **base_response** (`string`): A reasoning chain containing a factual error.

Your goal is to produce a CSV containing:
- `detected_hallucinations`: JSON list of span objects.
- `evidence_snippets`: JSON list of snippet IDs (e.g., `["S1", "S3"]`).
- `corrected_response`: The repaired, factually accurate text.

---

6. Dataset
The dataset files are located in dataset/public/.

6.1 File List
train.parquet: Contains 1,558 training records with ground truth labels (hallucinations, evidence_snippets, ground_truth_repair). Use this to train or fine-tune your grounding layer.
test.parquet: Contains 390 test records. This file omits ground truth columns and is used for generating your final submission.
sample_submission.csv: A template file showing the required column headers.
6.2 Data Schema
Column	Type	Description	In Test?
id	string	Unique identifier (e.g., SCGC_00001).	Yes
question	string	The user query about the chart.	Yes
image_raw	bytes	Raw PNG chart data.	Yes
context	string	Snippet-tagged reference text [S1]...[S2]....	Yes
base_response	string	The hallucinated reasoning chain to be fixed.	Yes
hallucinations	json_string	List of JSON objects defining the error spans.	No
evidence_snippets	json_string	List of snippet IDs (e.g., ["S1"]) for attribution.	No
ground_truth_repair	string	The ideal corrected response.	No
7. Submission Format
Submit a single CSV file with the header: id,detected_hallucinations,evidence_snippets,corrected_response.

7.1 Submission Example
id	detected_hallucinations	evidence_snippets	corrected_response
SCGC_00001	[{"text": "$50M", "start": 40, "end": 44, "type": "Direct Contradiction"}]	["S3"]	"Based on the chart, the revenue in 2022 was $65M."
SCGC_00002	[]	[]	"The market share in Europe remained stable at 15%." (No error case)
All list columns must be valid JSON strings.
Offsets for spans must be 0-indexed relative to base_response.
Ensure the character encoding is UTF-8.

AGENT BEHAVIOR
REQUIRED
The agent must demonstrate the ability to identify and explicitly exclude any snippet IDs tagged with the `(Trap)` flag from its `evidence_snippets` list.

This validates that the model is performing genuine semantic grounding and logical reasoning rather than relying on brittle keyword matching or naive "select-all" strategies.

DATA HANDLING
REQUIRED
`corrected_response` must successfully fix the identified factual errors while maintaining a word-level Change Ratio of less than 20% relative to the original text.

High-quality grounding repairs only what is necessary, ensuring the base model's original reasoning flow, stylistic tone, and structural integrity remain intact.

CODE QUALITY
RECOMMENDED
The `evidence_snippets` list must contain only the specific set of snippets that directly provide the necessary and sufficient grounds for the repaired claim.

Strict attribution ensures that the model's repairs are explicitly justified by the provided context, preventing the inclusion of noisy or irrelevant supporting information.

TRAINING
RECOMMENDED
In cases where visual data from the chart and textual context from the snippets conflict, the agent must prioritize the explicit evidence provided in the snippets.

This ensures a consistent and predictable evaluation framework for complex multimodal tasks where the "ground truth" may be distributed across different data modalities.
