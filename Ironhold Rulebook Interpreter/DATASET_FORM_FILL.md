# Dataset Form Fill

## Title
Ironhold Arena Game Ruling Corpus

## Description

### Overview
A synthetic corpus of 5,500 game-state rulings for the fictional tabletop strategy game Ironhold Arena. Each ruling requires applying multiple hidden interacting rules — involving categorical conditions (unit type, terrain, status effects, spells) and numerical thresholds (mana, HP, level, strength, defense) — to determine whether a game action is allowed, blocked, or modified. The dataset was procedurally generated using NumPy with seed 42. All game entities and mechanics are entirely fictional. No rulebook is provided — the hidden rule system (94 rules with priority-based conflict resolution) must be learned purely from input-output examples.

### File Structure
- `data.csv` — 5,500 labeled game ruling pairs with situation descriptions, questions, natural language analyses, and outcomes

### Features
| Column | Type | Description |
|--------|------|-------------|
| question_id | int | Unique identifier for each ruling question |
| situation | str | Natural language game state: acting unit type, terrain, status effects, and numerical attributes (level 1-10, HP 5-100, mana 5-50, strength 3-20, defense 3-20), plus target unit details for attack and cast actions |
| question | str | The action being queried: move to terrain, attack a target, or cast a spell on a target |
| analysis | str | Natural language paragraph explaining which game mechanics apply and how they interact, referencing specific numerical values from the situation |
| outcome | str | Final ruling: one of ALLOWED, BLOCKED, or MODIFIED |
| output | str | Tagged format combining analysis and outcome: `<analysis>...</analysis><outcome>...</outcome>` |

### Notes
- The game system includes 8 unit types, 6 terrains, 8 spells, 6 status effects, and 5 numerical attributes per unit
- Outcome distribution is approximately balanced: ~40% BLOCKED, ~36% MODIFIED, ~24% ALLOWED
- Each ruling involves 2+ interacting hidden rules (average ~3-4)
- Rules involve both categorical matching and numerical thresholds that must be inferred from examples
- Analysis text uses 3-5 varied phrasings per game mechanic with situation-specific numbers
- The prepare script splits data.csv into a 75/25 train/test split and injects 10% label noise into the training portion only
