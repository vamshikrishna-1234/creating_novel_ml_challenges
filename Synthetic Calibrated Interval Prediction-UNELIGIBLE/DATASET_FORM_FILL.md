## Overview

This dataset is fully synthetic, generated programmatically using NumPy with a fixed random seed (42) for full reproducibility. It simulates monitoring data from a fictional industrial reactor facility. Each sample represents a single reactor run characterized by 30 observable measurements and a continuous output variable (yield).

The dataset contains 50,000 reactor-run records. Each record has 13 continuous sensor measurements, 5 categorical process configuration codes (integer-coded), 5 post-processing residual measurements, and 7 cross-correlation metrics computed from pairs of sensors. The target variable (`yield_output`) is a continuous scalar representing the process yield for that run.

The underlying relationship between inputs and yield is non-linear, involving multi-way interactions and threshold effects. Critically, the observation noise is **not constant** — it varies across different operating regimes. Some sensor combinations produce highly predictable yields, while others operate in regimes where yield outcomes are inherently noisy. Approximately 8% of records have additional stochastic perturbation reflecting measurement anomalies.

The dataset also includes `noise_sigma`, the true standard deviation of the noise applied to each sample. This enables calibration analysis against the hidden noise structure.

## File Structure

- `data.csv` — 50,000 reactor-run records (50,000 rows, 32 columns). Each row represents one reactor run with its sensor readings, process configuration, derived metrics, the process yield, and the noise standard deviation.
