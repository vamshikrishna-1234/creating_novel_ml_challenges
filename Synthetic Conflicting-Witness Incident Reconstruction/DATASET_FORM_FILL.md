## Overview

This dataset is fully synthetic, generated programmatically by `generate.py` using NumPy with a fixed random seed (42) for full reproducibility. It simulates a fictional industrial safety reporting system where multiple independent observers provide conflicting accounts of the same incident, and a separate investigative process establishes the ground-truth structured incident report.

The dataset contains 8,000 synthetic incidents. Each incident has a ground-truth structured report consisting of 6 categorical fields (actor, action, location, time period, severity, contributing factor). For each incident, 4–6 independent observers produced free-text statements describing what they witnessed using varied sentence structures. Observers have hidden field-specific reliability profiles: the same observer may accurately report the location but misidentify the actor. Observers frequently disagree on 1–4 fields. Red-herring sentences within statements embed decoy entity codes from unrelated incidents, making naive extraction unreliable. Approximately 12% of incidents are genuinely ambiguous, where the ground truth itself was influenced by uncertainty.

The intended prediction task is **structured incident reconstruction from conflicting observer accounts**: given a set of disagreeing observer statements for an incident, predict the 6-field ground-truth structured report. The central challenge is resolving field-level disagreements across observers without knowing which observer is reliable on which field.

## File Structure

- `ground_truth.csv` — Ground-truth structured incident reports (8,000 rows)
- `statements.csv` — Observer free-text statements (~40,000 rows, 4–6 per incident)

## Features

**ground_truth.csv**

| Column | Type | Description |
|--------|------|-------------|
| incident_id | int | Unique incident identifier |
| actor | string | The individual involved (1 of 30 coded identifiers) |
| action | string | The action that occurred (1 of 20 coded identifiers) |
| location | string | Where the incident occurred (1 of 15 coded identifiers) |
| time_period | string | When the incident occurred (1 of 8 coded identifiers) |
| severity | string | Incident severity level (1 of 5 coded identifiers) |
| contributing_factor | string | Primary contributing factor (1 of 12 coded identifiers) |

**statements.csv**

| Column | Type | Description |
|--------|------|-------------|
| incident_id | int | Incident being described |
| witness_idx | int | Observer index within this incident |
| witness_role | string | Coded observer role identifier |
| statement | string | Free-text observer account of the incident |

## Notes

- Each incident has exactly one ground-truth structured report.
- Each incident has 4–6 observer statements.
- Observers disagree on 1–4 fields per incident based on hidden reliability profiles.
- Statements are written in varied formats (8+ templates). Red-herring sentences embed decoy codes from other incidents.
- All data is synthetically generated with a fixed random seed (42) for reproducibility.
- No real individuals, incidents, or observations are represented.
