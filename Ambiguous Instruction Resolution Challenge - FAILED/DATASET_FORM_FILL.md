# Dataset Description

## Overview

This dataset is fully synthetic. It simulates workplace communication scenarios where a natural language instruction contains linguistic ambiguity. The dataset was created to benchmark NLP systems on pragmatic language understanding — specifically, the ability to resolve ambiguities that arise naturally in professional communication such as pronoun references, scope of quantifiers, prepositional phrase attachment, elliptical constructions, quantifier–negation interaction, and temporal event ordering.

Each row represents a self-contained workplace scenario involving named employees, departments, and documents. The scenario provides context, followed by an ambiguous instruction. Four candidate interpretations are provided (A through D), exactly one of which is the correct resolution given the scenario context.

The dataset contains 20,000 rows covering six types of linguistic ambiguity in roughly equal proportions. It was generated programmatically using Python with controlled randomization (seed 42). No real individuals, organizations, or events are represented.

Some rows undergo additional obfuscation: person names may be replaced with anonymous codes (Person-1, Person-2, etc.), department names may be replaced with generic labels (Dept-A, Dept-B, etc.), scenario sentences may be reordered, option wording may be paraphrased, and irrelevant distractor sentences may be inserted into the scenario.

## File Structure

- `data.csv` — The complete dataset (20,000 rows, 8 columns)

## Features

| Column | Type | Description |
|--------|------|-------------|
| id | int | Unique row identifier (0 to 19999) |
| scenario | string | A 3–6 sentence workplace scenario providing context. Contains entity names (or anonymized codes), department names (or anonymized codes), document references, and contextual clues needed to resolve the ambiguity. May contain red-herring sentences. |
| instruction | string | A single sentence containing linguistic ambiguity (pronoun reference, scope, attachment, ellipsis, quantifier, or temporal). |
| option_a | string | Candidate interpretation A — one possible resolution of the instruction. |
| option_b | string | Candidate interpretation B — one possible resolution of the instruction. |
| option_c | string | Candidate interpretation C — one possible resolution of the instruction. |
| option_d | string | Candidate interpretation D — one possible resolution of the instruction. |
| answer | string | The correct option label: one of A, B, C, or D. |

## Notes

- All scenarios are independent; no row references another row.
- Answer labels are balanced: roughly 25% each of A, B, C, D.
- The correct interpretation can only be determined by reading and understanding the scenario context — it is not derivable from the instruction or options alone.
- Obfuscation (name anonymization, sentence reordering, paraphrasing, red herrings) is applied deterministically to a subset of rows.
- No external data sources, real persons, or copyrighted material are used.
