## Overview

This dataset is fully synthetic, generated programmatically by `generate.py` using NumPy with a fixed random seed (42) for full reproducibility. It simulates a fictional service engagement platform where requestors browse pools of candidate profiles and select exactly one per engagement session. The intended prediction task is **session-level selection prediction**: given a requestor, a pool of candidates, and per-candidate interaction signals, predict which candidate the requestor will select.

The dataset comprises three entity types — **candidates** (500 service providers with category assignments and numeric quality attributes), **requestors** (3,000 clients with category preferences and numeric preference weights), and **sessions** (15,000 engagement events, each linking one requestor to 6–14 candidates with per-candidate behavioral signals). A fourth file records the ground-truth selection for each session.

The underlying selection function combines category alignment between requestor preferences and candidate categories, latent feature compatibility (interaction of requestor weights with candidate attributes), positional ordering effects, candidate prominence, and stochastic session-type variation (normal, impulsive, passive browsing). This produces non-trivial selection patterns that resist simple heuristic prediction. Key modeling challenges include: (1) the selection depends on bilateral compatibility between requestor and candidate features, not on either profile alone; (2) interaction signals are noisy proxies of the latent compatibility; (3) approximately 1.4% of requestors in the test split have no profile entry, requiring cold-start handling; and (4) the candidate pool size varies per session (6–14), creating variable-length input.

## File Structure

- `interactions.csv` — Per-candidate interaction records across all 15,000 sessions (~150,260 rows)
- `candidates.csv` — Candidate profile catalog (500 rows)
- `requestors.csv` — Requestor profile catalog (3,000 rows)
- `selections.csv` — Ground truth: which candidate was selected per session (15,000 rows)

## Features

**interactions.csv**

| Column | Type | Description |
|--------|------|-------------|
| session_id | int | Unique engagement session identifier |
| requestor_id | int | Requestor who conducted this session |
| candidate_id | int | Candidate profile viewed in this session |
| position | int | Display position of this candidate within the session pool |
| engagement | float | Engagement intensity signal for this candidate interaction |
| dwell_time | float | Time spent reviewing this candidate profile (seconds) |
| browse_depth | float | Depth of profile exploration (0.0 to 1.0) |
| revisit | int | Whether the requestor revisited this candidate (0 or 1) |

**candidates.csv**

| Column | Type | Description |
|--------|------|-------------|
| candidate_id | int | Unique candidate identifier |
| category | int | Candidate category assignment (0–19) |
| cf_1 through cf_6 | float | Numeric candidate quality attributes |
| popularity | float | Candidate prominence score (0.0 to 1.0) |

**requestors.csv**

| Column | Type | Description |
|--------|------|-------------|
| requestor_id | int | Unique requestor identifier |
| pref_cat_1 | int | Primary category preference |
| pref_cat_2 | int | Secondary category preference |
| pref_cat_3 | int | Tertiary category preference |
| rw_1 through rw_6 | float | Numeric requestor preference weights |
| cat_strength | float | Strength of category preference influence |

**selections.csv**

| Column | Type | Description |
|--------|------|-------------|
| session_id | int | Session identifier |
| selected_candidate_id | int | The candidate selected in this session |

## Notes

- Each session has exactly one selected candidate.
- A requestor may appear in multiple sessions.
- Candidates may appear across many sessions.
- All data is synthetically generated with a fixed random seed (42) for reproducibility.
- No real individuals, services, or transactions are represented.
