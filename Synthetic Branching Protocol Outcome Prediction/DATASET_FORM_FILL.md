## Overview

This dataset is fully synthetic, generated programmatically using NumPy with a fixed random seed (42) for full reproducibility. It simulates a fictional biochemical processing system where coded procedural protocols are executed under varying initial conditions to produce structured outcomes.

The dataset contains 24,000 execution records from 8,000 unique procedural protocols. Each protocol is a directed acyclic graph (DAG) of 8–15 steps described in free text using coded terminology (fictional reagent names, tool identifiers, and phase descriptors). Protocols contain conditional branches where the execution path depends on the current state of internal variables, as well as steps with side effects that modify variables used by later conditional checks. Some steps are no-operations disguised as real processing steps, and some are low-impact interference steps.

Each protocol is executed under 3 different sets of initial conditions, producing potentially different outcomes due to the branching structure. A hidden latent variable (not exposed in the data) additionally influences branch resolution at 1–2 decision points per protocol, meaning the same visible protocol and conditions can produce different outcomes depending on this unobserved factor. Approximately 7% of executions result in ambiguous outcomes due to underspecified branch conditions.

The intended prediction task is **structured outcome prediction from procedural text and initial conditions**: given a protocol description and a set of initial variable values, predict the 4-field execution outcome.

## File Structure

- `data.csv` — All execution records (24,000 rows, 12 columns). Each row represents one execution of a procedural protocol under a specific set of initial conditions. Contains the protocol identifier, condition set index, the full protocol text description (free-text with shuffled step ordering), 5 initial condition variables, and 4 structured outcome fields (terminal state, primary product, byproduct class, process status).

## Notes

- Each protocol has exactly 3 execution records (one per condition set).
- Step ordering within protocol_text is shuffled — steps reference each other by step-ID (e.g., S01, S02), not by position.
- All reagent, tool, and phase names are fictional coded identifiers.
- A hidden latent variable influences branch resolution, creating irreducible uncertainty.
- All data is synthetically generated with a fixed random seed (42) for reproducibility.
- No real substances, processes, or laboratory procedures are represented.
