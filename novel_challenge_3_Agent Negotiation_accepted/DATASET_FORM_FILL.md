# Dataset Description

## Overview

This dataset is synthetically generated (no real users, agents, or transactions). It represents **multi-turn negotiation transcripts** between two fictional agents (buyer and seller) trading invented goods. Each row is one complete negotiation. The task is to predict the **deal outcome** (4-class classification).

**Dataset Summary:**

- **Multi-turn dialogue:** Each transcript contains 3–16 turns with speaker tags ([BUYER], [SELLER], [SYSTEM]) separated by `|||`. Turns include English negotiation phrases mixed with invented protocol tokens from a fictional protocol language ("Nexari", e.g. NEXO:OFFER, NEXO:COUNTER, NEXO:REJECT).
- **Outcome classes:** `deal_accepted`, `deal_rejected`, `counter_proposed`, `timeout` — determined by negotiation patterns (concession rate, protocol token sequences, offer gap trajectory, stall behavior).
- **Structured features:** `num_turns` (int) and `sector` (5 categories: energy, defense, biotech, logistics, manufacturing) accompany each transcript.
- **Reproducibility:** Data is produced by `generate.py` (default seed 42, 30,000 rows). Python standard library only; no external sources.

**Current raw dataset stats:**

- Total rows: 30,000
- Columns: id, transcript, num_turns, sector, label
- Label distribution: deal_accepted 8,949 (30%), deal_rejected 7,410 (25%), counter_proposed 8,396 (28%), timeout 5,245 (17%)
- Sectors: roughly balanced (~6,000 each)
- Turns: min 3, max 16, mean 8.3
- Transcript length: 184–1,371 characters

## File Structure

- `data.csv` — labeled data (id, transcript, num_turns, sector, label)

No other files in the raw dataset. The challenge prepare script splits this into public/private splits.

## Features

| Column     | Type   | Description |
|------------|--------|-------------|
| id         | int    | Unique identifier |
| transcript | string | Multi-turn negotiation dialogue. Turns separated by `\|\|\|`. Contains English + Nexari protocol tokens (NEXO:OFFER, NEXO:COUNTER, etc.). Speaker tags: [BUYER], [SELLER], [SYSTEM]. |
| num_turns  | int    | Number of dialogue turns in the transcript. Range 3–16. |
| sector     | string | Industry sector. One of: energy, defense, biotech, logistics, manufacturing. |
| label      | string | Deal outcome. One of: deal_accepted, deal_rejected, counter_proposed, timeout. |

## Notes

- **Reproducibility:** Run `python generate.py --output data.csv --seed 42 --size 30000` to regenerate.
- **Nexari protocol:** Invented protocol tokens (NEXO:OFFER, NEXO:COUNTER, NEXO:ACCEPT, NEXO:REJECT, NEXO:HOLD, NEXO:TIMEOUT, etc.) are embedded in turns. Their sequences carry signal about the outcome.
- **One label per row:** Each id appears exactly once. No duplicate transcripts in the generator output.
- **Class imbalance:** Intentionally slightly imbalanced (timeout is ~17% minority). Grading uses a frequency-weighted macro F1 that rewards handling the minority class well.
- **Train/test split:** The challenge prepare script splits by row (80/20) with stratification and a fixed seed; no overlapping ids between train and test.
