```markdown
# Dataset Description

## Overview

This dataset is synthetically generated (no real users or systems). It represents code-switched customer-support-style text: each row is one complaint mixing English with tokens from a fictional constructed language ("Verani"). The task is to predict resolution time in hours (continuous regression).

**Dataset Summary:**

- **Code-switched text:** English phrases plus Verani tokens (e.g. shaluma, voshka, bretal); resolution time is partly driven by code-switch density and embedded signals.
- **Structured signal:** Priority tags [P1]–[P5] and product category appear in the data; both are useful for prediction.
- **Reproducibility:** Data is produced by the script `generate.py` (default seed 42). No external sources; Python standard library only.

**Current raw dataset stats:**

- Total rows: 28,000
- Columns: id, text, category, target
- Target (resolution hours): range 0.5–168, two decimal places
- Categories: 8 (billing, authentication, data-pipeline, ui-frontend, api-gateway, storage, notifications, analytics)

## File Structure

- `data.csv` — labeled data (id, text, category, target)

No other files in the raw dataset. The challenge prepare script splits this into public/private splits.

## Features

| Column   | Type   | Description |
|----------|--------|-------------|
| id       | int    | Unique identifier |
| text     | string | Code-switched complaint text (English + Verani tokens). May contain priority tags [P1]–[P5]. |
| category | string | Product/area category. One of: billing, authentication, data-pipeline, ui-frontend, api-gateway, storage, notifications, analytics. |
| target   | float  | Resolution time in hours. Range 0.5–168. Two decimal places. |

## Notes

- **Reproducibility:** Run `python generate.py --output data.csv --seed 42 --size 28000` to regenerate.
- **Verani:** Fictional language tokens are mixed into sentences; resolution time is partly driven by code-switch density and embedded signals (priority, severity phrasing).
- **One label per row:** Each id appears exactly once with one target. No duplicate texts in the generator output.
- **Train/test split:** The challenge prepare script splits by row (80/20) with a fixed seed; no overlapping ids between train and test.
```
